- red-team's "record the curl" is structural → DB-doc convention (`4911243`), held across D26.
- Rule-reversal-sync (PROMOTED ×3) + whole-file/ledger-subsection refinement (PROMOTED ×2) — both in CLAUDE.md; NOT a mechanical grep.

- `state-lift-vs-widget.x` (`cfbfe7f`) RESOLVED — dynamic AppBar title read frozen `widget.task`; fix `late _task`+setState.
- RPC-write shape proven-low-risk (D26 COMPLETE 4/4; D36 = SOLE write path). Spend attention on per-entity deltas.
- `toRpcParams()`↔RPC-arity seam = recurring RPC-write failure (×2 `1e7574d`,`258cb6c`); inverse DEFAULT-wipe near-miss (`3b0468a`). One `database.md` #2 line covers both.
- Read-only entity must gate EVERY write affordance incl. STATE-DEPENDENT (RULE CANDIDATE ×2 `258cb6c`,`adab034`). learner-PROPOSED → `design-principles.md`.
- Shared test fakes → `test/support/fakes.dart` (D42, `a08c199`): share at ≥2-file dup, specials stay local; paydown of the reconstructing-fake flag. Rename trap: private fake NAME ≠ one behavior across files — map by BEHAVIOR.
